# 🔒 Security Policy\n\n## Unterstützte Versionen\n\nWir bieten Sicherheitsupdates für die folgenden Plugin-Versionen an:\n\n| Version | Unterstützt          |\n| ------- | -------------------- |\n| 1.8.x   | ✅ Vollständig       |\n| 1.7.x   | ✅ Kritische Fixes   |\n| 1.6.x   | ⚠️ Nur kritische     |\n| < 1.6   | ❌ Nicht unterstützt |\n\n## Sicherheitslücke melden\n\n### 🚨 Responsible Disclosure\n\nWenn Sie eine Sicherheitslücke im KaPlan Gottesdienste Plugin gefunden haben, melden Sie diese bitte **privat** und **nicht** über öffentliche Issues.\n\n### 📧 Kontakt für Sicherheitsmeldungen\n\n**Email**: Über [jlsoftware.de](http://www.jlsoftware.de) Kontakt aufnehmen  \n**Antwortzeit**: Normalerweise innerhalb von 48 Stunden  \n**Sprachen**: Deutsch, Englisch  \n\n### 📋 Was sollten Sie in Ihrem Bericht angeben?\n\n1. **Beschreibung der Sicherheitslücke**\n   - Art des Problems (XSS, SQL Injection, etc.)\n   - Betroffene Plugin-Version(en)\n   - Schweregrad der Lücke\n\n2. **Schritte zur Reproduktion**\n   - Detaillierte Anweisungen\n   - Proof-of-Concept (falls möglich)\n   - Screenshots oder Videos\n\n3. **Potenzielle Auswirkungen**\n   - Welche Daten könnten betroffen sein?\n   - Welche Benutzer sind gefährdet?\n\n4. **Ihre Umgebung**\n   - WordPress-Version\n   - PHP-Version\n   - Plugin-Version\n   - Browser (falls relevant)\n\n### 🛡️ Was wir garantieren\n\n- **Vertraulichkeit**: Ihre Meldung wird vertraulich behandelt\n- **Schnelle Reaktion**: Antwort innerhalb von 48 Stunden\n- **Anerkennung**: Sie werden (mit Ihrer Erlaubnis) in den Release Notes erwähnt\n- **Kein rechtliches Vorgehen**: Sofern Sie verantwortungsvoll handeln\n\n## 🔍 Sicherheitspraktiken\n\n### Was wir tun\n\n- ✅ **Input-Validierung**: Alle Benutzereingaben werden validiert\n- ✅ **Output-Escaping**: HTML-Ausgaben werden escaped (`esc_html`, `esc_url`)\n- ✅ **Sichere API-Aufrufe**: HTTPS und Zertifikatsvalidierung\n- ✅ **Code-Reviews**: Alle Änderungen werden überprüft\n\n### Bekannte Sicherheitsüberlegungen\n\n1. **API-Schlüssel**: Das Plugin überträgt KaPlan-Zugangsdaten über HTTPS\n2. **XSS-Schutz**: Alle dynamischen Inhalte werden escaped\n3. **Input-Sanitization**: Parameter werden validiert und bereinigt\n\n## 🚀 Sicherheitsupdates\n\n### Automatische Updates\n\nSicherheitsupdates werden priorisiert und über das automatische Update-System verteilt:\n\n1. **Kritische Sicherheitslücken**: Sofortige Patches\n2. **Mittlere Sicherheitsprobleme**: Updates innerhalb von 7 Tagen\n3. **Niedrige Sicherheitsverbesserungen**: Reguläre Updates\n\n### Update-Benachrichtigungen\n\n- WordPress Admin zeigt verfügbare Updates an\n- Sicherheitsupdates werden als \"wichtig\" markiert\n- Release Notes enthalten Sicherheitsinformationen\n\n## 📊 Schweregrad-Einstufung\n\n### 🔴 Kritisch\n- Remote Code Execution\n- SQL Injection\n- Authentifizierungs-Umgehung\n- Privileg-Eskalation\n\n### 🟠 Hoch\n- Cross-Site Scripting (XSS)\n- Cross-Site Request Forgery (CSRF)\n- Informationsleck sensitiver Daten\n\n### 🟡 Mittel\n- Informationsleck nicht-sensitiver Daten\n- Denial of Service (DoS)\n- Bypass von Sicherheitsmaßnahmen\n\n### 🟢 Niedrig\n- Informationsleck öffentlicher Daten\n- Minimale Sicherheitsverbesserungen\n\n## 🛠️ Für Entwickler\n\n### Sicherheits-Checklist\n\nBevor Sie Code beitragen:\n\n- [ ] Alle Eingaben validiert?\n- [ ] Alle Ausgaben escaped?\n- [ ] CSRF-Schutz implementiert?\n- [ ] Berechtigung geprüft?\n- [ ] Sichere API-Aufrufe?\n- [ ] Keine Geheimnisse im Code?\n- [ ] Security-Tests durchgeführt?\n\n---\n\n## 🙏 Danksagung\n\nWir danken der Security-Community für ihre Hilfe beim Schutz unserer Benutzer.\n\n---\n\n*Letzte Aktualisierung: September 2025*
